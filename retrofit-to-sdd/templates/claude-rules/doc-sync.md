# 文档同步规范（防漂移）

> 适用于本仓库一切代码 / 需求 / 架构变更。AI 代理与人类协作者都须遵守。完整流程见 `docs/CONTRIBUTING.md`。

## 1. 文档即代码（强制）

改代码的**同一次变更**里，必须把受影响的文档改到一致。**文档没同步 = 变更未完成。**

## 2. 改了什么 → 必须同步哪个文档

| 你改了 | 必须同步 |
|---|---|
| 需求 / 范围 / 验收标准 | `docs/PRD.md` |
| 架构决策（新技术 / 新模式 / 推翻旧决策） | 新增 `docs/adr/NNNN-*.md`（见 §3） |
| 模块 / 数据模型 / 机制 / 依赖流向 | `docs/ARCHITECTURE.md` |
| REST 接口 | `docs/API.md` |
| 任何用户可见变更 | `CHANGELOG.md` 未发布段 |

## 3. ADR 不可变 + 取代

- 已接受的 ADR **不改正文**；决策变了写**新 ADR 取代**旧的，旧 ADR 仅把状态改为"已被 ADR-NNNN 取代"。
- 编号递增、永久保留、不删除、不复用。

## 4. 单一真源

同一事实只在一处权威描述，禁止复制散落到多个文档；需要时**引用**而非复制。

## 5. 禁止

- 禁止只改代码不改文档（造成漂移）。
- 禁止编辑已接受 ADR 的决策正文（应另写 ADR 取代）。
- 禁止把易朽过程稿（实施计划 / 探索笔记）写进 `docs/` 入库；它们留 `.tmp/`。
